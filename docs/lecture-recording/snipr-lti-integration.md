# SNIpR LTI 1.3 Integration Guide

> **Version:** v1.2
> **Last Updated:** March 31, 2026
> **Status:** Production Ready

## Overview

SNIpR supports LTI 1.3 integration with learning management systems (LMS) like ILIAS and Moodle. This allows instructors to embed lecture recordings directly into their courses, with automatic authentication and role-based access control.

### LTI 1.3 Flow

```
┌──────────────┐     LTI Launch      ┌──────────────┐
│   ILIAS/     │ ──────────────────► │    SNIpR     │
│   Moodle     │  (with course ID)   │   (Rust)     │
│   (LMS)      │ ◄────────────────── │              │
└──────────────┘   Video Player      └──────────────┘
                        │
                        ▼
                 ┌──────────────┐
                 │  Keycloak    │
                 │   (SSO)      │
                 └──────────────┘
```

## Prerequisites

- SNIpR deployed and configured (see [SNIpR Setup](./snipr-setup.md))
- Keycloak configured with LTI client
- LMS (ILIAS 7+ or Moodle 4.4+) with LTI 1.3 support

## Keycloak LTI Client Configuration

### Step 1: Create LTI Client

1. Log into Keycloak Admin Console
2. Navigate to **Realm Settings** → **Clients**
3. Create new client: `snipr-lti`
   - **Client ID:** `snipr-lti`
   - **Client protocol:** `openid-connect`
   - **Access type:** `Confidential`
   - **Valid redirect URIs:**

     ```
     https://ilias.opendesk.example.com/snipr/lti/launch/*
     https://moodle.opendesk.example.com/mod/lti/launch/*
     ```

   - **Web origins:**

     ```
     https://ilias.opendesk.example.com
     https://moodle.opendesk.example.com
     ```

   - **Service accounts enabled:** `ON`
   - **Client authentication:** `ON`

### Step 2: Configure LTI Claims

In Keycloak, navigate to **Clients** → `snipr-lti` → **Mappers**:

Add the following mappers:

| Name | Mapper Type | Token Claim | Claim Name | Value |
|------|-------------|-------------|------------|-------|
| `lti_version` | Constant | `lti_version` | `https://purl.imsglobal.org/vocab/lis/v2/lti/version` | `LTI1p3` |
| `lti_role` | User Attribute | `lti_role` | `https://purl.imsglobal.org/vocab/lis/v2/membership` | `${role}` |
| `resource_link_id` | Constant | `resource_link_id` | `https://purl.imsglobal.org/vocab/lis/v2/resource/link/id` | `${course_id}` |
| `context_id` | Constant | `context_id` | `https://purl.imsglobal.org/vocab/lis/v2/course/context/id` | `${course_id}` |
| `context_title` | Constant | `context_title` | `https://purl.imsglobal.org/vocab/lis/v2/course/context/title` | `${course_name}` |
| `user_id` | User Attribute | `user_id` | `https://purl.imsglobal.org/vocab/lis/v2/user/id` | `${username}` |
| `name` | User Attribute | `name` | `name` | `${firstName} ${lastName}` |
| `given_name` | User Attribute | `given_name` | `given_name` | `${firstName}` |
| `family_name` | User Attribute | `family_name` | `family_name` | `${lastName}` |
| `email` | User Attribute | `email` | `email` | `${email}` |

### Step 3: Configure Role Mapping

Create a role mapper to translate Keycloak roles to LTI roles:

**Instructor Role:**

```
https://purl.imsglobal.org/vocab/lis/v2/membership#Instructor
```

**Student Role:**

```
https://purl.imsglobal.org/vocab/lis/v2/membership#Learner
```

**Mapping in SNIpR values.yaml:**

```yaml
lti:
  roles:
    instructor: https://purl.imsglobal.org/vocab/lis/v2/membership#Instructor
    student: https://purl.imsglobal.org/vocab/lis/v2/membership#Learner
```

### Step 4: Copy Client Credentials

After creating the client, copy:

- **Client ID:** `snipr-lti`
- **Client Secret:** (generated by Keycloak)

Update `helmfile/environments/default/secrets.yaml.gotmpl`:

```yaml
secrets:
  snipr:
    ltiClientSecret: "${SNIPR_LTI_SECRET}"
```

## ILIAS Integration

### Step 1: Install LTI 1.3 Plugin

ILIAS requires the LTI 1.3 plugin:

```bash
# Download and install LTI 1.3 plugin
cd /var/www/ilias/plugins
git clone https://github.com/elan-ev/ilias-lti13-plugin.git
composer install --no-dev
```

### Step 2: Configure LTI Provider in ILIAS

1. Log into ILIAS as Administrator
2. Navigate to **Administration** → **Plugins** → **LTI 1.3 Provider**
3. Click **Add LTI 1.3 Provider**
4. Configure:
   - **Provider Name:** `SNIpR Lecture Recording`
   - **Client ID:** `snipr-lti`
   - **Client Secret:** (from Keycloak)
   - **Authorization Endpoint:** `https://keycloak.opendesk.example.com/realms/opendesk/protocol/openid-connect/auth`
   - **Token Endpoint:** `https://keycloak.opendesk.example.com/realms/opendesk/protocol/openid-connect/token`
   - **JWKS Endpoint:** `https://keycloak.opendesk.example.com/realms/opendesk/protocol/openid-connect/certs`
   - **Redirect URI:** `https://snipr.opendesk.example.com/api/lti/launch`
5. Save configuration

### Step 3: Add SNIpR to Course

1. Navigate to desired course in ILIAS
2. Click **Add Repository Object** → **External Tool (LTI)**
3. Configure:
   - **Title:** `Lecture Recordings`
   - **Provider:** `SNIpR Lecture Recording`
   - **Launch Type:** `Inline (iframe)`
   - **Course ID:** (auto-populated)
4. Save

### Step 4: Verify Integration

1. As instructor, verify you can see all recordings for the course
2. As student, verify you can only view recordings (no upload/edit)
3. Test deep linking: Click a recording → should open in ILIAS iframe

## Moodle Integration

### Step 1: Install LTI 1.3 Support

Moodle 4.4+ includes native LTI 1.3 support. No additional plugins required.

### Step 2: Configure External Tool in Moodle

1. Log into Moodle as Administrator
2. Navigate to **Site administration** → **Plugins** → **Activity modules** → **External tool**
3. Click **Add external tool**
4. Configure:
   - **Tool name:** `SNIpR Lecture Recording`
   - **Tool URL:** `https://snipr.opendesk.example.com/api/lti/launch`
   - **Consumer key:** (not used in LTI 1.3)
   - **Shared secret:** (not used in LTI 1.3)
   - **Platform URL:** `https://moodle.opendesk.example.com`
   - **Platform OID:** `moodle-snipr`
5. Save

### Step 3: Add SNIpR to Course

1. Navigate to desired course in Moodle
2. Turn **Edit mode** ON
3. Click **Add an activity or resource** → **External tool**
4. Configure:
   - **External tool:** `SNIpR Lecture Recording`
   - **Launch container:** `Inline (iframe)`
   - **Activity ID:** (auto-populated)
5. Save and return to course

### Step 4: Verify Integration

1. As instructor, verify you can see all recordings for the course
2. As student, verify you can only view recordings
3. Test deep linking: Click a recording → should open in Moodle iframe

## Role-Based Access Control

### Instructor Permissions

- View all recordings in course
- Upload new recordings
- Edit recording metadata (title, description, tags)
- Delete recordings
- Manage transcription settings
- Export recordings

### Student Permissions

- View recordings (based on course enrollment)
- Search recordings by transcript
- Bookmark/favorite recordings
- Download recordings (if enabled)

### Configuration

```yaml
lti:
  roles:
    instructor: https://purl.imsglobal.org/vocab/lis/v2/membership#Instructor
    student: https://purl.imsglobal.org/vocab/lis/v2/membership#Learner
    teachingAssistant: https://purl.imsglobal.org/vocab/lis/v2/membership#TeachingAssistant
```

## Deep Linking

SNIpR supports LTI 1.3 Deep Linking, allowing instructors to select specific recordings to embed in course materials.

### Deep Linking Flow

1. Instructor clicks **Add External Tool** in ILIAS/Moodle
2. Selects **SNIpR Deep Linking**
3. SNIpR presents list of recordings for the course
4. Instructor selects one or more recordings
5. LTI launch returns with `content_items` payload
6. LMS embeds selected recordings in course page

### Deep Linking Configuration

```yaml
lti:
  deepLinking:
    enabled: true
    acceptTypes:
      - iframe
      - link
    autoLaunch: true
```

## Troubleshooting

### LTI Launch Fails

**Symptom:** "Invalid token" or "JWT validation failed"

**Check:**

```bash
kubectl logs -l component=snipr -c snipr-api | grep -i "lti"
```

**Solution:** Verify Keycloak JWKS endpoint is accessible and client credentials are correct

### Role Not Recognized

**Symptom:** User sees wrong permissions (e.g., student sees instructor UI)

**Check:**

```bash
kubectl logs -l component=snipr -c snipr-api | grep -i "role"
```

**Solution:** Verify LTI role mapping in Keycloak mappers

### Course ID Not Passed

**Symptom:** SNIpR shows all recordings instead of course-specific

**Check:**

```bash
kubectl logs -l component=snipr -c snipr-api | grep -i "context_id"
```

**Solution:** Verify LTI launch includes `context_id` claim
