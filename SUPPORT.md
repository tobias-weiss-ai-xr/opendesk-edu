## 🐛 Reporting Issues / Probleme melden

### Bug Reports / Fehlerberichte

**Before reporting / Vor dem Melden:**

1. Check if the issue is already reported / Überprüfen Sie, ob das Problem bereits gemeldet wurde
2. Check the documentation / Überprüfen Sie die Dokumentation
3. Try the latest version / Probieren Sie die neueste Version

**How to report / Wie man meldet:**

1. Go to [Issues](https://github.com/opendesk-edu/opendesk-edu/issues)
2. Click "New Issue"
3. Choose "Bug Report"
4. Fill in the template completely

**Include / Einschließen:**

- Environment details (Kubernetes, Helm versions)
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs

### Feature Requests / Feature-Anfragen

**Before requesting / Vor dem Anfragen:**

1. Check existing feature requests / Überprüfen Sie bestehende Feature-Anfragen
2. Consider if it fits the project scope / Überlegen Sie, ob es in den Projektumfang passt

**How to request / Wie man anfragt:**

1. Go to [Issues](https://github.com/opendesk-edu/opendesk-edu/issues)
2. Click "New Issue"
3. Choose "Feature Request"
4. Describe the problem and proposed solution

## ❓ FAQ / Häufige Fragen

### Deployment / Bereitstellung

<details>
<summary><strong>I'm getting Helm errors during deployment. What should I do? / Ich erhalte Helm-Fehler während der Bereitstellung. Was soll ich tun?</strong></summary>

1. Verify your Helm version (3.x required)
2. Check your values files for syntax errors
3. Run `helmfile -e dev template` to see rendered output
4. Check the [Troubleshooting Guide](./docs/troubleshooting.md) if available
5. Search [existing issues](https://github.com/opendesk-edu/opendesk-edu/issues)

</details>

<details>
<summary><strong>SSO login is not working. How do I debug this? / SSO-Login funktioniert nicht. Wie kann ich das debuggen?</strong></summary>

1. Check Keycloak logs: `kubectl logs -n opendesk deployment/keycloak`
2. Verify IdP configuration in values.yaml
3. Check service SP metadata
4. Verify network connectivity between services
5. Review the [SSO documentation](./docs/external-services.md)

</details>

<details>
<summary><strong>A service is not starting. What should I check? / Ein Dienst startet nicht. Was soll ich überprüfen?</strong></summary>

1. Check pod status: `kubectl get pods -n opendesk`
2. View pod logs: `kubectl logs -n opendesk <pod-name>`
3. Describe pod for events: `kubectl describe pod -n opendesk <pod-name>`
4. Check resource constraints (CPU/memory limits)
5. Verify persistent volume claims are bound

</details>

### Configuration / Konfiguration

<details>
<summary><strong>How do I customize the portal? / Wie kann ich das Portal anpassen?</strong></summary>

Portal customization is documented in [docs/theming.md](./docs/theming.md). Key areas:

- Logo and branding
- Color scheme
- Service tiles and categories
- Custom CSS

</details>

<details>
<summary><strong>How do I add custom services? / Wie füge ich benutzerdefinierte Dienste hinzu?</strong></summary>

1. Create a new Helm chart in `helmfile/charts/`
2. Add the release to `helmfile.yaml`
3. Configure SSO integration if needed
4. Add portal tile configuration
5. Follow the [Contributing Guide](./CONTRIBUTING.md) for PR submission

</details>
